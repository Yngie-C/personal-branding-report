## 대표 프로젝트

### 1. AI 추천 시스템 구축 (2022-2023)

**배경 & 문제 정의**
- 기존 추천 알고리즘의 클릭률이 업계 평균 대비 40% 낮음
- 사용자 이탈률 증가 (월 15% → 23%)
- 콜드 스타트 문제로 신규 유저의 30%가 첫 주에 이탈

**내 역할**
- 프로젝트 PM 및 오너
- 데이터 사이언티스트, ML 엔지니어, 백엔드 개발자 8명 팀 리드
- 비즈니스 메트릭 정의 및 성공 지표 설정

**접근 방법**
1. 문제 분석
   - 50명 사용자 인터뷰 진행 (정성 데이터)
   - 100만 건 클릭 로그 분석 (정량 데이터)
   - 경쟁사 벤치마킹 (Netflix, Spotify 등 5개사)

2. 솔루션 설계
   - Collaborative Filtering + Content-based 하이브리드 모델
   - A/B 테스트 프레임워크 구축 (10% 트래픽으로 시작)
   - 실시간 피드백 루프 시스템 설계

3. 실행 및 검증
   - MVP 4주 스프린트 3회 진행
   - 주간 데이터 리뷰 미팅 운영
   - 점진적 롤아웃 (10% → 50% → 100%)

**결과**
✅ 클릭률 62% 증가 (1.8% → 2.9%)
✅ 사용자 체류 시간 45% 증가 (평균 12분 → 17분)
✅ MAU 140% 성장 (50만 → 120만)
✅ NPS 점수 35 → 58 향상
✅ 연간 매출 기여도 $2.3M 추정

**배운 점**
- 완벽한 ML 모델보다 빠른 실험과 반복이 중요
- 데이터만으로는 부족, 사용자 맥락 이해가 핵심
- 점진적 롤아웃으로 리스크 최소화 가능

**관련 링크**
- 프로젝트 케이스 스터디: medium.com/@kimtech/ai-recommendation
- 컨퍼런스 발표 영상: youtube.com/watch?v=xxx
- 기술 블로그 포스트: tech.company.com/recommendation-system

### 2. B2B SaaS 플랫폼 0→1 런칭 (2021-2022)

**배경 & 문제 정의**
- 타겟: 중소기업 HR 담당자 (직원 50-500명 규모)
- 기존 솔루션: 복잡하고 비싸며 설정에 2주 이상 소요
- 기회: 연 $50M 시장에서 언더서빙된 세그먼트

**내 역할**
- 0→1 Product Lead
- 제품 비전 수립 및 Go-to-Market 전략 기획
- 초기 고객 10개사 직접 온보딩

**접근 방법**
1. 고객 발견 (Discovery)
   - 잠재 고객 인터뷰 80회
   - 섀도잉 8회 (실제 업무 관찰)
   - "Jobs to be Done" 프레임워크 적용
   - Pain Point Map 작성 및 우선순위화

2. MVP 정의
   - Must-have 기능 3가지만 선정
   - 설정 시간 목표: 30분 이내
   - Figma 프로토타입으로 사전 검증 (15개사)
   - Tech Stack 결정 (Next.js, Supabase, Vercel)

3. Build-Measure-Learn
   - 2주 스프린트로 12주 내 MVP 출시
   - Early Access 프로그램 운영 (베타 유저 25명)
   - 주간 사용 데이터 리뷰 및 즉각 개선
   - Feature Flag 활용한 점진적 기능 출시

4. Growth
   - Product-Led Growth 전략 (Self-serve 온보딩)
   - 바이럴 루프 설계 (초대 1건당 $50 크레딧)
   - 컨텐츠 마케팅 (SEO 최적화 블로그 20편)

**결과**
✅ 런칭 후 6개월 내 고객 50개사 확보
✅ 첫 해 ARR $500K 달성
✅ MRR Growth Rate 월평균 25%
✅ Net Dollar Retention 120%
✅ 고객 만족도 4.6/5.0 (NPS 65)
✅ 시리즈 A 투자 유치에 핵심 기여 ($3M)

**핵심 인사이트**
- "기능 많음 ≠ 좋은 제품", 핵심 가치에 집중
- 초기 10명의 팬을 만드는 게 100명의 무관심보다 중요
- PLG는 제품 자체가 마케팅, 온보딩 UX가 생명
- 고객 피드백을 듣되, 요청을 그대로 구현하지 않기

**미디어 & 인정**
- ProductHunt "Product of the Month" (2021.09)
- TechCrunch 기사 게재
- SaaStr 컨퍼런스 케이스 스터디 발표

### 3. 모바일 앱 리뉴얼 프로젝트 (2019-2020)

**배경**
- 기존 앱: 3년간 기술 부채 누적, 앱스토어 평점 3.2
- 주요 컴플레인: 느린 속도, 직관적이지 않은 UI, 잦은 크래시
- 비즈니스 임팩트: 유료 전환율 하락 (8% → 4%)

**내 역할**
- 프로덕트 오너 (PO)
- iOS/Android 개발자 6명, 디자이너 2명 팀
- 6개월 타임라인 관리

**프로세스**
1. 현재 상태 진단
   - 앱 크래시 로그 분석 (Firebase Crashlytics)
   - Hotjar 히트맵으로 사용자 행동 분석
   - 1on1 사용자 인터뷰 30회
   - 기술 부채 우선순위 매핑 (Engineering 팀과 협업)

2. 리뉴얼 전략
   - 점진적 개선 vs 전면 재작성 트레이드오프 분석
   - 결정: 하이브리드 접근 (Core는 재작성, 부가 기능은 점진 개선)
   - Design System 구축 (일관성 확보)
   - 성능 벤치마크 설정 (앱 실행 3초 이내, API 응답 500ms 이내)

3. 실행
   - Feature Flagging으로 구/신 버전 A/B 테스트
   - 베타 테스팅 프로그램 운영 (파워 유저 200명)
   - 크래시 프리 세션 95% 이상 달성 후 정식 출시
   - 단계별 롤아웃 (5% → 25% → 100%)

**결과**
✅ 앱스토어 평점 3.2 → 4.5 (Google Play 4.6)
✅ 크래시율 8% → 0.5%
✅ 앱 실행 속도 7초 → 2초
✅ 리텐션 D7: 25% → 48%, D30: 12% → 28%
✅ 유료 전환율 회복 (4% → 9%, 이전 대비 +12.5%)
✅ "Best of 2020" 앱스토어 선정

**도전과 극복**
- 도전: 개발 중 크리티컬 버그 발견으로 1개월 딜레이
- 극복: 스코프 조정 + 우선순위 재정렬로 타임라인 맞춤
- 교훈: 버퍼 타임 20% 항상 확보 필요

## 프로덕트 철학

### 1. 사용자 중심 사고
"데이터는 What을 말하고, 사용자는 Why를 말한다"
- 정량 데이터 (Analytics)와 정성 데이터 (Interview) 균형
- 사용자 공감 지도(Empathy Map) 작성 습관화
- 월 최소 5명 이상 고객과 대화

### 2. 임팩트 우선주의
"빠르게 실패하고, 현명하게 피봇하라"
- 80/20 법칙: 20%의 기능이 80%의 가치 창출
- 가설 → 실험 → 검증 → 학습 사이클 최소화
- Shipped > Perfect

### 3. 협업의 힘
"혼자 가면 빠르지만, 함께 가면 멀리 간다"
- 투명한 커뮤니케이션 (Weekly Product Update)
- 크로스펑셔널 존중 (개발/디자인/비즈니스)
- 데이터 기반 의사결정으로 주관 배제
